This is an example of applied PCA and Low rank Approximation

Here we will be interested in examing some kind of data set

prereq: [[PCA]], [[K Rank Approx]]

Here is the set up for the problem. Given a set of multi-demensional data that has been carefully processed (meaning that all the variances that are caused by unitary transformation has been voided), put them into a matrix XXX and each data object from the set is a column on the data matrix XXX.

Performs a SVD on the matrix, so that we obtain the decomposition of the matrix, wrritten as X=UΣVTX = U\Sigma V^TX=UΣVT.

Perform PCA analysis and choose the the first ppp singular values that best explain the variance of the data. Which will provides us with a lower rank approximation using the matrices U^Σ^V^T\hat{U}\hat{\Sigma}\hat{V}^TU^Σ^V^T

Now assume that we are given a new data that doesn't belong to the columns of XXX.

Then the rank approximation for that data using the basis of our decomposition will be XXTyXX^TyXXTy where y is the column vector representing the data that we don't know.

And the output of the expression will be an reconstruction of the data using the given basis.

Interpretation

The singular values(diagonal of the matrices) explains variance of the data set and each of the basis in UUU is associated with a given singular value. The first singular value is associated with the first column in UUU and it explains the most variance on our data set, and the second one corresponds to the second column in matrix UUU and the amount of variance explained by it is second to all the other singular values and so on.

The matrix UUU is the eigen data set, the columns of the matrix UUU is used to reconstruct data in the training set, or data that are not in the training set.

Note that, in the case where the raining set mannage to span the whole dimension of the data set, then the UUU is going to span Rm\mathbb{R}^mRm and in that case, UUU can be used to represent everything.

The rows of the matrix VVV (columns the matrix VTV^TVT) is like the finger print of that data that is in the matrix XXX, it's the best combinations of singular values and its corresponding columns in UUU such that it can be used to reconstruct the columns in the original matrix as much as possible.

For a given data that is not in the trainning set, the finger print, with the principal conponents inside of it, it's simply UTyU^TyUTy.

Now suppose that the given yyy is in the data set, and it's very close to one of the data object, then we can do the following:

UTyU^Ty UTy

Will gives us some the projection of the data object yyy onto the ortho basis. And then, among all the rows of the matrix VVV find the viv_ivi​ such that Σvi\Sigma v_iΣvi​ is closest to UTyU^TyUTy, and then, we use the index iii to get the closest match of data in XXX to the given data yyy.

In addition, we can also do this under the most must singnificant basis for the data set instead of the full data set, this will reduce the computational recourses immensely.
