Here we are going to discuss in details about the LU decomposition, so that we have a better understanding of it.

This part of the book looks at some of the stuff behind the idea of an LU decomposition and find some exploit of it, so that it's easier to understand.

Most of this is coming from the Trefethen's book, numerical Linear Algebra.

Introduction

The idea behind the LULULU decomposition is to encode the process of Gaussian Eliminations into 2 matrix, the LLL as an lower triangular matrix and the UUU as an upper triangular matrix.

The LU decomposition applies to square matrix AAA, both complex and real, let's assume that A∈Cm×mA\in \mathbb{C}^{m \times m}A∈Cm×m

Lm−1Lm−2⋯L2L1A=UL_{m - 1}L_{m - 2} \cdots L_{2}L_{1}A = U Lm−1​Lm−2​⋯L2​L1​A=U

Where each LLL represents a row eliminations process during the Gaussian Elimination (For now assume no pivoting is needed during the elimination process). And then we would like to find the inverse of those sequence of row operations matrix.

L=L1−1L2−1L3−1⋯Lm−2−1Lm−1−1L = L_1^{-1} L_2^{-1} L_3^{-1} \cdots L^{-1}_{m - 2}L^{-1}_{m - 1} L=L1−1​L2−1​L3−1​⋯Lm−2−1​Lm−1−1​

and then The matrix A=LUA = LUA=LU.

Note: This algorithm is used to compute the determinant of a matrix in matlab codes.

A General Formulation

For this part, we are still going to be assuming that the matrix A∈Cm×mA\in \mathbb{C}^{m\times m}A∈Cm×m. And we are looking at one of the row operation matrix, LkL_kLk​

The LkL_kLk​ matrix is the matrix that operates on the kthk^{th}kth column of the matrix, and aims to eliminates the entry zero for all rows that are below kkk.

The LkL_kLk​ matrix experience the following structure:

(Lk)i,i=1∀  1≤i≤m(L_k)_{i, i}= 1 \quad \forall \; 1 \leq i \leq m (Lk​)i,i​=1∀1≤i≤m

This is for the diagonal entries, and for the row operational elements, it's gonna looking like:

(Lk)i,k=xj,kxk,k∀  k<j≤m(L_k)_{i, k} = \frac{x_{j, k}}{x_{k, k}} \quad \forall \; k < j \leq m (Lk​)i,k​=xk,k​xj,k​​∀k<j≤m

Ok... too much indexing, let's try if we can leverage the power of the matrix by representing it as something else.

Let the vector lll be negative of the kthk^{th}kth column of the matrix LkL_kLk​, but with the kkk element set to 1 instead of zero.

In addition, the block structure of the matrix allows us to easily multiply together a sequences of the inverse matrices, more specifically:

L=L1−1L2−1⋯Lm−2−1Lm−1−1 L = L_1^{-1}L_2^{-1} \cdots L^{-1}_{m-2}L^{-1}_{m-1} L=L1−1​L2−1​⋯Lm−2−1​Lm−1−1​

And this is the formula to get the Lower Triangular matrix LLL for the LU decomposition. With pivoting.

LU Decomposition Partial Pivoting

When eliminating the columns to get the matrix UUU we might consider choosing the row where on the kkk th column we can have the element such that it's the largest.

We achieve this using a permutation matrix at each step of the decomposition, during this discussion, without much loss of generality, we will be assuming the the matrix we want to decompose is 4 by 4.

And this will be giving us the following:

L3P3L2P2L1P1A=UL_3P_3L_2P_2L_1P_1A = U L3​P3​L2​P2​L1​P1​A=U

Now, spoiler:

L3P3L2P2L1P1A=L3′L2′L1′P3P2P1L_3P_3L_2P_2L_1P_1A = L_3'L_2'L_1'P_3P_2P_1 L3​P3​L2​P2​L1​P1​A=L3′​L2′​L1′​P3​P2​P1​

Where the prime notation on the lower triangular matrix denotes matrices of the following form:

L3′=L3L_3' = L_3 L3′​=L3​

L2′=P3L2P3−1L_2' = P_3L_2P_3^{-1} L2′​=P3​L2​P3−1​

L1′=P3P2L1P2−1P3−1L_1' = P_3P_2L_1P_2^{-1}P_3^{-1} L1′​=P3​P2​L1​P2−1​P3−1​

Notice that, the matrix P3P_3P3​, P2P_2P2​ are not acting on the first column of the matrix L1L_1L1​, it leaves it alone, and this is recursively true for the sub-matrices.

So in general, we will have the following:

(Lm−1′⋯L2′L1′)(Pm−1⋯P2P1)A=U(L_{m-1}'\cdots L_2'L_1')(P_{m - 1}\cdots P_2P_1)A = U (Lm−1′​⋯L2′​L1′​)(Pm−1​⋯P2​P1​)A=U

Where LkL_kLk​ is defined by:

Lk′=Pm−1⋯Pk+1LkPk+1−1⋯Pm−1−1L_k' = P_{m - 1}\cdots P_{k+1}L_kP_{k + 1}^{-1} \cdots P_{m - 1}^{-1} Lk′​=Pm−1​⋯Pk+1​Lk​Pk+1−1​⋯Pm−1−1​

And then in the end, we would love to figure out the inverse of all the stuff that multiplied on the LHS of the matrix AAA and this is going to give us something like this:

PA=LUPA = LU PA=LU

Where:

L=(Lm−1′Lm−2′⋯L2′L1′)−1L = (L'_{m -1}L'_{m - 2}\cdots L'_2L'_1)^{-1} L=(Lm−1′​Lm−2′​⋯L2′​L1′​)−1

Algorithm 21.1

This algorithm with partial pivoting is from Trefethen's Book.

function [L, U, P] = LuDecompose(A)
    % This function performs a LU matrix decomposition with the pivoting.
    % This is the  for midterm of AMATH 584. 
    [m, n] = size(A); 
    if m ~= n
        error("Matrix must be squared. ")
    end
    U = A; L = eye(m); P = eye(m);

    for K = 1: n - 1
        [MaxEntry, I] = max(abs(U(K:end, K)));
        if MaxEntry < 1e-16
           error("Matrix Hardly Invertible");
        end
        I  = I + K - 1;
        [U(K, K:m), U(I, K:m)] = swap(U(K, K:m), U(I, K:m));
        if K >= 2
            [L(K, 1: K - 1), L(I, 1: K - 1)] = ... 
                swap(L(K, 1: K - 1), L(I, 1: K - 1));
        end
        
        [P(K, :), P(I, :)] = swap(P(K, :), P(I, :));
        
        for J = K + 1: m
           L(J, K) = U(J, K)/U(K, K); 
           U(J, K:m) = U(J, K:m) - L(J, K)*U(K, K: m);
        end
    end
    U = triu(U);
    function [a, b] = swap(b, a)
    end
end

from numpy import identity, array, triu,set_printoptions, argmax
from numpy.random import rand
from math import isclose
eye = identity
arr = array

def lu_decompose(A):
    """
        Function will performs a LU decomposition on the given matrix
    :param A:
        A invertible matrix.
    :return:
        P, L, U such that PA = LU
    """
    M, N = A.shape
    if M != N:
        raise Exception("Matrix must be squared. ")
    U = A.copy().astype("float64"); L = eye(M); P = eye(M)
    for K in range(N - 1):
        I = argmax(abs(U[K:, K])) + K
        assert not isclose(U[I, K], 0), "Hardly Invertible Matrix"
        U[[K], K:], U[[I], K:] = U[[I], K:], U[[K], K:]
        if K >= 1: L[[K], :K], L[[I], :K] = L[[I], :K], L[[K], :K]
        P[[K], :], P[[I], :] = P[[I], :], P[[K], :]
        # Cancellations
        L[K + 1:, K] = U[K + 1:, K]/U[K, K]
        for J in range(K + 1, M):
            U[J, K:] = U[J, K:] - L[J, K]*U[K, K:]
    return P, L, triu(U)

This is a very well-optimized algorithm, the PPP permutations matrices are never represented as an actual matrix multiplication! But the operations will be felt by these matrices because of the use of <--> on these matrices.

How is the LLL matrix constructed? This part of the algorithm is the most fascinating part.

L1−1P1L_1^{-1} \quad P_1 L1−1​P1​

The is the first 2 row operational matrix and the row permutation matrix.

L2−1P2L1−1P1P2L_2^{-1}P_2L_1^{-1} \quad P_1P_2 L2−1​P2​L1−1​P1​P2​

Notice that, the P2 Matrix actually acts on both L1−1L_1^{-1}L1−1​ and L2−1L_2^{-1}L2−1​ matrix, and that is kinda dope.

L3−1P3L2−1P2L1−1P3P1P2L_3^{-1}P_3L_2^{-1}P_2L_1^{-1} \quad P_3P_1P_2 L3−1​P3​L2−1​P2​L1−1​P3​P1​P2​

The matrix (L1)(L_1)(L1​) is going to feel the aggregate all the affects from the permutations matrix P3P2P1P_3P_2P_1P3​P2​P1​ and the row operation matrices L2−1L3−1L_2^{-1}L_3^{-1}L2−1​L3−1​

And that is what the algorithm is doing.
