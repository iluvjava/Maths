[[Line Search Gradient Descend]]

Major Reference: <Numerical Optimization>

Intro

Given a descend direction, we search for the best step size into the direction that assures a balance between sufficient objective decrease and sufficient stepsizes.

In the relevant file above, we had steepest descend and the armijo line search.

Here we introduce the: Strong Wolfe Condition.

List of parameters:

c1,c2,α,x(k),p,∇[f]c_1, c_2, \alpha, x^{(k)}, p, \nabla[f] c1​,c2​,α,x(k),p,∇[f]

  • α\alphaα is the step size that we want to determine, easily.
  • 0<c1<c2<10 < c_1 < c_2 < 10<c1​<c2​<1 are 2 of the small constant that determines the behaviors of the line search algorithm.
  • x(k)x^{(k)}x(k), the point are currently at.
  • ppp, the search direction, we assume that this is a search direction vector that brings decreases to the objective function.
  • ∇[f]\nabla[f]∇[f] The gradient operator.

Wolfe Conditions

f(x(k)+αp)≤f(x(k))+c1α∇[fT](x(k))p∇[f](x(k)+αp)Tp≥c2∇[fT](x(k))p\begin{aligned} f(x^{(k)} + \alpha p) \le& f(x^{(k)}) + c_1 \alpha \nabla [f^T](x^{(k)})p \\ \nabla [f](x^{(k)} + \alpha p)^T p \ge& c_2 \nabla [f^T](x^{(k)}) p \end{aligned} f(x(k)+αp)≤∇[f](x(k)+αp)Tp≥​f(x(k))+c1​α∇[fT](x(k))pc2​∇[fT](x(k))p​

  • The first conditions prevent it from over stepping, stepping pass the local minima of a convex region (If there is any local minima and it's a convex region)

  • The second condition will assert some kind of convexity constraint. It also makes sure that the step size is not too small. For sufficiently small stepsize, say α=0\alpha = 0α=0, it decreases the objective too much compare to the relaxed second ray, c2pT∇[f](x(k))c_2p^T\nabla[f](x^{(k)})c2​pT∇[f](x(k))[1]

The second condition is sometimes wrote as:

∣pT∇[f](x(k)+αp)∣−c2∣pT∇[f](x(k))∣≤0\left| p^T \nabla[f](x^{(k)} + \alpha p) \right| - c_2 \left| p^T \nabla[f](x^{(k)}) \right| \le 0 ∣∣​pT∇[f](x(k)+αp)∣∣​−c2​∣∣​pT∇[f](x(k))∣∣​≤0

  • Along the search direction, the gradient of the objective function is at least, decreasing, and increasing by a bit is allowed if c2<1c_2 < 1c2​<1.
  • The absolute value ignore the direction information.

Intuitive Understanding People should have

The stepsize will have to stay below the first ray and the gradient at the new point must be less steep compare to the second ray. (The gradient is at least, decreasing).

Basic Line Search Algorithm

Armijo line search introduced in: [[Line Search Gradient Descend]] it's a simple implementations that eliminates over stepping and ensures sufficient objective decrease by terminating the forloop once sufficient decrease is satisfied.

Sophisticated Line Search Algorithm

Another approach will be more involved, it seeks to keep both wolfe conditions without too much evaluations of the gradient/value on the function. But it's actually not too hard. Let's take a look.

What it tries to do is:

It tries to prob for a step size as long as possible, and then decide on an window before it starts searching. (This part is algorithm 3.5, 3.6 in the Numerical Optimization textbook)

Let's define a scalar function:

ϕ(t)=f(x(k)+tp(k))\phi(t) = f(x^{(k)} + t p^{(k)}) ϕ(t)=f(x(k)+tp(k))

The function achored at the previous guess of the solution and prob into the direction of p(k)p^{(k)}p(k) with distance ttt.
