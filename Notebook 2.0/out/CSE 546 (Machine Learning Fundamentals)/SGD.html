[[Coordinate Descend]] We continue the saga for gradient descend, and here, we will be looking at another popular gradient descend algorithm used for machine learning.

Intro

We can't compute the loss easily, because we need to sum up the loss for all the samples in the training set.

This is too huge to compute in the general sense. For just one gradient update on the parameters of the models.

Similar to the spirit of Coordinate descend, where we only focus on one parameter at a time, instead of doing full gradient on all data, we are going to just draw one sample.

And then, we update the gradient using just that one sample.

Jensen's Inequality

It's statistics mixed with convex analysis.

for any random Z∈RdZ\in\mathbb{R}^dZ∈Rd, and convex function ϕ:Rd↦R\phi: \mathbb{R}^d \mapsto \mathbb{R}ϕ:Rd↦R we have: ϕ(E[Z])≤E[ϕ(Z)]\phi(\mathbb{E}[Z])\le \mathbb{E}[\phi(Z)]ϕ(E[Z])≤E[ϕ(Z)].

The function value is always less than the line/hyperplane that interpolate points on the domain.

This inequality is used to help to prove this inequality:

E[l(wˉ)−l(w∗)]≤1T∑t=1TE[l(wt)−l(w∗)]\mathbb{E}\left[ l(\bar{w}) - l(w_*) \right] \le \frac{1}{T}\sum_{t = 1}^{T} \mathbb{E}\left[l(w_t) - l(w_*)\right] E[l(wˉ)−l(w∗​)]≤T1​t=1∑T​E[l(wt​)−l(w∗​)]

Which appeared in the previous analysis on the boundeness of the Stochastic Gradient Descend Algorithm.

The intuition behind the Jensen's inequality is simple, take a look at the convex definition of a function:

f(tX1+(1−t)X2)≤f(X1)+(1−t)f(X2)f(tX_1 + (1 -t)X_2)\le f(X_1) + (1 - t)f(X_2) f(tX1​+(1−t)X2​)≤f(X1​)+(1−t)f(X2​)

And if we just let the variable X1,X2X_1, X_2X1​,X2​ be drawn from the domain of the function and randomly dran, then:

ϕ(E[X])≤E[ϕ(X)]\phi(\mathbb{E}\left[X\right]) \le \mathbb{E}\left[\phi(X)\right] ϕ(E[X])≤E[ϕ(X)]

Mini-Batch SGD

Instead of optimizing on each sample, in practice, will update on batches. This will produce smaller variance on the model. At the same time, the process is highly parallelizable.
