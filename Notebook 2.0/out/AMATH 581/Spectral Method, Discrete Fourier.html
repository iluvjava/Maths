Derivative With FFT

Using the properties of the Fourier Transform, we can perform a transformation in the Frequencies domain to take the derivative of the given function.

This is how we use the DFT to find the derivative of some kind of periodic data:

L  = 5;
N  = 2^8;
x2 = linspace(-L/2, L/2, N + 1);  % Query points. 
x  = x2(1:N);  % Trim off the last point because of periodic conditions. 

u = exp(-x.^2) % cos(x).*exp(-x.^2/25);
U_Fourier = fft(u);

k = (2*pi/L)*(-N/2: N/2 - 1);  % The lower infinite and upper inifity on the fourier transform. 
k = fftshift(k);  % Must shift this or it's not working. 

figure(1);
plot(x, u); hold on;

% Taking derivative on the Fourier Space it's gonna be like: 

dU_F  = i.*k.*U_Fourier;
dU2_F = (i.*k).^2.*U_Fourier;
dU    = ifft(U_Fourier);
plot(x, real(dU));
plot(x, real(ifft(dU_F)));
plot(x, real(ifft(dU2_F)));

Notice:

Notice the usage of the fftshift command, it has to be done for the derivative coefficients or the fft functions itself to.

F(f∣ζ)=∫−∞∞exp⁡(−2πiζx)f′(x)  dx\mathcal{F}(f|\zeta) = \int_{-\infty}^{\infty} \exp\left( -2\pi i \zeta x \right)f'(x)\; dx F(f∣ζ)=∫−∞∞​exp(−2πiζx)f′(x)dx

Where kkk is replaced by 2πζ2\pi \zeta2πζ. This form is closer to the Discrete, FFT algorithm. It's stated here because it can created confusion on the multiplier of the derivative put into the discrete Fourier Transform.

And when it's DFT, the multiplier for derivative on the Fourier Domain is gonna look like:

  • Assume periodic boundary conditions on the interval [−L/2,L/2][-L/2, L/2][−L/2,L/2]
  • Assuming Spatial Discretization with NNN number of evenly distributed points. Then the derivative coefficients under the Fourier Domain is going to be expressed as:

k=2πL(N2:N2−1)k = \frac{2\pi}{L}\left(\frac{N}{2}:\frac{N}{2}- 1\right) k=L2π​(2N​:2N​−1)

And before applying this vector to the Fourier Domain, we need to shift (fftshift) the lower frequencies domain to the center so it matches the format with the Fourier Series.

Comment:

It's tempting to do the integration via dividing by the multiplier ikikik, but please take note that we will have a divide by zero problem when doing that.

2D DFT

Assuming Periodic transform conditions, consider an M×NM\times NM×N grid of data points, the 2d DFT can phrased by the following formula:

F(k,l)=∑m=0M−1∑n=1N−1f(m,n)exp⁡(−i2π(kmM+lnN))F(k, l) = \sum_{m = 0}^{M - 1}\sum_{n = 1}^{N - 1} f(m, n)\exp\left( -i2\pi\left( \frac{km}{M} + \frac{ln}{N} \right) \right) F(k,l)=m=0∑M−1​n=1∑N−1​f(m,n)exp(−i2π(Mkm​+Nln​))

Ok, looks complicated but it's actually something like this:

F(k,l)=∑m=0M−1(exp⁡(−2iπkmM)∑n=0N−1f(m,n)exp⁡(−i2πlnN))F(k, l) = \sum_{m = 0}^{M - 1} \left( \exp \left( -2i\pi\frac{km}{M} \right) \sum_{n = 0}^{N - 1}f(m, n)\exp \left( -i2\pi\frac{ln}{N} \right) \right) F(k,l)=m=0∑M−1​(exp(−2iπMkm​)n=0∑N−1​f(m,n)exp(−i2πNln​))

And the operations are nested, and it will be something like this:

F(k,l)=∑m=0M−1(exp⁡(−2iπkmM)Fl(f(m,_)∣m))F(k, l) = \sum_{m = 0}^{M - 1} \left( \exp \left( -2i\pi\frac{km}{M} \right) F_l(f(m, \_)|m) \right) F(k,l)=m=0∑M−1​(exp(−2iπMkm​)Fl​(f(m,_)∣m))

Ok, let's phrase this in term of matrix instead.

The 1d DFT is a matrix vector multiplications and can be phrased in the following way:

F=WNfF = W_Nf F=WN​f

Where the W is the DFT matrix, and WWW will be N×NN\times NN×N in our case.

Let's use this to make the above expression better and see what is going on here:

F(k,l)=∑m=0M−1(exp⁡(−2iπkmM)(WNf[m,_])l)F(k, l) = \sum_{m = 0}^{M - 1} \left( \exp \left( -2i\pi\frac{km}{M} \right) (W_Nf[m,\_])_l \right) F(k,l)=m=0∑M−1​(exp(−2iπMkm​)(WN​f[m,_])l​)

F(k,l)=(WM(WNf[M=m,N=n])l)kF(k, l) = (W_M (W_Nf[M = m,N = n])_l)_k F(k,l)=(WM​(WN​f[M=m,N=n])l​)k​

The (k,l)(k, l)(k,l) is gotten by:

  • Take 1D DFT on all the columns and take out the lll th columns.
  • Take 1D DFT on the lll th columns and take out the kkk th element from that lll th column.

Cool, now let fff be a M×NM\times NM×N data matrix, and write it like f[i,j]f[i, j]f[i,j], then the above 2D DFT should be able to summarized as:

WM((WNfT)T)W_M((W_Nf^T)^T) WM​((WN​fT)T)

Partial XY in Fourier Domain

Sometimes, when solving the PDEs, we apply Fourier Transform on both the x direction, and the y direction. And analytically, this is as simple as multiplying it by ikxik_xikx​, ikyik_yiky​, however when doing it with DFT, we need to know how to do it.

Advection Diffusion Example

As we see in the [[Advection Diffusion]] example, we need to solve a linear system to get the stream function before doing the time-stepping.

So the math goes something like this:

∂x2ϕ+∂y2ϕ=ω\partial_x^2 \phi + \partial_y^2 \phi = \omega ∂x2​ϕ+∂y2​ϕ=ω

Take Fourier transform on ∂x\partial_x∂x​ and ∂y\partial_y∂y​ on both side we have:

ω^=−kx2ϕ^−ky2ϕ^\widehat{\omega} = -k_x^2 \widehat{\phi} - k_y^2 \widehat{\phi} ω=−kx2​ϕ​−ky2​ϕ​

ϕ^=−ω^kx2+ky2\widehat{\phi} = -\frac{\widehat{\omega}}{k_x^2 + k_y^2} ϕ​=−kx2​+ky2​ω​

After this, we can use the DFT to compute the right hand side and then take the inverse Fourier to get the stream function.
